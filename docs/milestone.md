# SeeForMe 项目里程碑报告

## 已完成

### 后端（FastAPI）
- 用户认证：注册 / 登录 / JWT（access + refresh token）
- 双角色体系：求助者（seeker）/ 志愿者（volunteer）
- 求助单生命周期：发布 → 接单 → 完成 / 取消
- 文件上传：图片 / 音频 / 视频（5 / 10 / 50 MB 限制）
- 志愿者回复 & 求助者收到通知
- 评价反馈（rating）
- 举报 & 拉黑
- 接口测试（21 个测试用例通过）
- CI/CD（GitHub Actions）

### 前端（Expo 跨平台）
- 欢迎 / 注册 / 登录 / 角色选择页
- 求助者：发起求助、大厅、消息、个人页
- 志愿者：大厅（浏览 & 接单）、任务列表、消息、个人页

## 需要优化的地方

1. **安全**：`SECRET_KEY` 硬编码默认值；CORS 允许所有来源 `*`，上线前必须收敛。
2. **无障碍体验**：前端尚未针对屏幕阅读器（VoiceOver / TalkBack）做专项适配；缺少语音导航、大字模式。
3. **AI 集成**：STT / TTS / 图像描述是核心功能，目前全是占位符，优先级最高。
4. **实时性**：求助者发单后只能轮询，志愿者接单无实时推送，体验差。
5. **志愿者通知**：接口已定义但未实现，志愿者无法收到新求助提醒。
6. **错误提示**：前端对网络失败、权限不足等场景缺少对用户友好的提示。

## 如何获得第一位视觉障碍用户

### 线上渠道（成本低、速度快）
1. **残障社群**：加入微信 / QQ 里的视障者群，直接介绍产品，邀请内测。
2. **小红书 / 知乎**：发布产品故事，关键词「盲人」「视觉障碍」「无障碍」，吸引有需求的用户主动联系。
3. **B 站**：录制一段 demo 视频（展示求助 → 志愿者响应全流程），投放给无障碍相关 UP 主或博主。

### 线下渠道（信任度高、反馈深）
1. **盲校 / 盲人按摩店**：直接到附近盲校或盲人按摩中心拜访，1 对 1 演示，收集真实痛点。
2. **残联 & 助残协会**：联系当地残联，申请成为公益合作项目，借助其渠道触达视障群体。
3. **视障辅助技术培训班**：参加或赞助手机无障碍培训活动，现场演示。

### 降低使用门槛
- 先提供**网页版**（无需安装 App），降低初次体验门槛。
- 提供**语音引导式注册**，让无法看清屏幕的用户也能独立完成。
- 设置专属「内测反馈群」，及时响应首批用户问题，建立信任。
